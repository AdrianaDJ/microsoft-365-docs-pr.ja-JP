# <a name="admin-support-for-microsoft-managed-desktop"></a>Microsoft マネージドデスクトップの管理者サポート

Microsoft マネージドデスクトップの管理ポータルを使用して、サポートチケットまたはフィードバック要求を Microsoft に送信できます。 サポート要求は常にフィードバック送信に優先されます。 サポート要求は、「[重要度の定義」テーブル](#sev)で説明されているように、重要度に基づいてトリアージおよび管理されます。 フィードバックが確認され、要求された応答があります。 

>[!IMPORTANT]
>アプリパッケージ、デバイス、セキュリティ、およびその他の管理者の[連絡先が設定](../get-started/add-admin-contacts.md)されていることを確認してください。 管理者の連絡先が構成されていない場合、これらの領域のいずれにもサポート要求を送信することはできません。

**サポートリクエストを送信するには**
1. [Microsoft Managed Desktop 管理ポータル](https://aka.ms/mwaasportal)にサインインします。 
2. [**サポート要求**] で、[ **+ 新しいサポートチケット**] を選択します。
3. 必要なヘルプに一致する**サポート要求の種類**を選択します。 次の表は、オプションの概要を示しています。 
4. **重大度レベル**を選択します。 詳細については、「 [Support request severity definitions](#sev)」を参照してください。 

サポート要求の種類 | 使用する状況
--- | ---
事例 | 変更またはセキュリティインシデントが広範囲に及ぶ影響など、調査するために Microsoft Managed Desktop Operations Team が必要です。
情報の要求 | ネットワーク、プロキシ構成、VPN システム、証明書の有効期限の変更を計画しているか、またはサービスに関する何らかの情報を必要とするだけです。 組織内で変更を伝える場合は、Microsoft Managed Desktop Operations Team からの応答を強くお勧めします。
変更要求 | 更新プログラムグループ間でのデバイスの移動などの変更を行うには、Microsoft Managed Desktop Operations Team が必要です。

## <a name="support-request-severity-definitions"></a>サポート要求の重要度の定義

最初の応答時間は、Microsoft Managed Desktop 技師がお客様との連絡を行い、サポート要求の作業を開始したときに、サポート要求を提出した時点からの期間です。 最初の応答時間は、要求の重要度に基づいて、要求のビジネスへの影響に応じて異なります。

重大度レベル  | お客様の状況 |  最初の応答時間   | 期待される顧客対応
--- | --- | --- | ---
**重要度: 重大な影響** |  **ビジネスへの重大な影響**<br><br>お客様のビジネスには、サービスの著しい損失または低下が発生し、すぐに対応する必要があります。<br><br>**アプリケーション互換性に対する主要な影響**<br><br>お客様のビジネス全体が、重要な機能がクラッシュまたは失われているために財政的な影響を受ける。 | イニシャル: 1 時間 <<br>更新プログラム:60 分<br>24時間365日利用可能 | 重要度 A を選択すると、問題が重大なビジネスへの影響を与え、サービスの重大な損失と低下が発生することを確認できます。 <br><br>この問題はすぐに対応する必要があります。また、解決するまで Microsoft teams で24時間365日の運用を継続することをお約束します。そうでない場合は、Microsoft は重要度をレベル B に減らすことができます。<br><br> また、Microsoft が正確な連絡先情報を持っていることも確認します。

追加情報:
- **サポート言語**-すべてのサポートは英語で提供されています。
- **重大度レベルの変更**-お客様が適切なリソースまたは応答を提供できない場合は、microsoft が問題解決の作業を継続できるようにすることができます。 
- **営業時間**-ほとんどの国では、勤務時間は 9:00 AM から 5:00 PM、太平洋標準時までです。
- **アプリケーション互換性**-アプリケーション互換性の問題について考慮する場合は、Windows または Office の以前のバージョンと現在のバージョンの間で、同じバージョンのアプリケーションの reproduceable エラーが発生している必要があります。 アプリケーションの互換性の問題を解決するには、Microsoft がお客様の連絡ポイントを使用する必要があります。 個人は、Fast Track teams を直接操作して問題を調査および解決する必要があります。
- **顧客の応答時間**お客様が予想される応答要件を満たすことができない場合、Microsoft は、少なくとも1つの重要度レベル (重大度 C) に対して要求をダウングレードします。お客様がアクションの要求に応答しない場合、Microsoft は最終要求の48時間以内にサポートリクエストを軽減し、終了します。

## <a name="provide-feedback"></a>フィードバックの提供

フィードバックをお送りください。また、それを使用して管理者のサポートを向上させることができます。

チケットが**緩和**された状態または**解決済み**の状態になったら、その特定の問題を使用して、フィードバックを共有できます。 これを行うには、管理ポータルの [**サポート要求**] ページに移動します。 特定のチケットを選択します。 右側に表示されるフライインで、[**フィードバック**] タブを選択し、要求された情報を入力します。 フィードバックフォームに個人情報を含めないように注意してください。 プライバシーの詳細については、 [Microsoft のプライバシー](https://privacy.microsoft.com/privacystatement)に関する声明を参照してください。
