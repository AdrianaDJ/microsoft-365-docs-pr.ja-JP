## <a name="device-wipe-with-factory-reset"></a>工場出荷時リセットを持つデバイスの消去

管理されたデスクトップの運用チームは、工場出荷時に操作を行います可否する必要がある Microsoft 管理されたデスクトップ管理のデバイスをリセットします。これは、デバイスを別の従業員に提供する必要がある場合、または従業員が退職した場合に役立ちます。 

いくつかの要件があります。

- お客様のテナント管理者は、サービス リクエストを送信する必要があります。
- デバイスのコンピューター名にする必要があります。
- ユーザー アカウントがリセット前に、Azure AD である必要があります。

デスクトップ運用チームの管理を行います。

- Intune でデバイス名を検索します。
- デバイスを工場出荷時のリセット コマンドを送信します。

>[!NOTE]
>工場出荷時リセットする前に、Azure AD からユーザー アカウントを取り外さないでください。ユーザーは Azure AD でない場合は、Intune が工場出荷時リセット コマンドをデバイスに送信できません。 

OOBE は、デバイスが起動し、すべてのプレインストールされているアプリケーションと設定が再度適用します。デバイスのユーザーは、初期情報をもう一度セットアップを提供する必要があります。 

デバイスがリセットされたときに組織のほかの人に付けることが。デバイスの以前のユーザーのデータや企業データになります。次のユーザーは、新しい Microsoft 管理デスクトップ デバイスと前の人と同じプロセスを経てください。

BitLocker は、このプロセスでのデータ セキュリティの重要なコンポーネントです。BitLocker 暗号化を使用する Microsoft 管理デスクトップ デバイスのドライブ上のデータは、デバイスを工場出荷時リセットを適用した後にもセキュリティで保護されたままです。ドライブに存在していたすべてのデータはデバイスの次のユーザーに利用できません。詳細については、 [BitLocker の概要](https://docs.microsoft.com/windows/security/information-protection/bitlocker/bitlocker-overview)を参照してください。

詳細については、[工場出荷時のデバイスのリセット](https://docs.microsoft.com/intune/devices-wipe#factory-reset-a-device)を参照してください。
