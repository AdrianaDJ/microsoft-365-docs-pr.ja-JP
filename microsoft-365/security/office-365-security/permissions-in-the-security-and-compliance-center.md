# <a name="permissions-in-the-security--compliance-center"></a>セキュリティ/コンプライアンス センターのアクセス許可

コンプライアンス センター&、デバイスの管理、データ損失防止、電子情報開示、保持などのコンプライアンス タスクを実行するユーザーにアクセス許可を付与できます。 これらのユーザーは、明示的にアクセス権を付与されたタスクのみを実行できます。 セキュリティ センターからセキュリティ &アクセスするには、ユーザーは、グローバル管理者か、1 つ以上のセキュリティ グループの役割グループの&する必要があります。

コンプライアンス センターのセキュリティ &許可は、役割ベースのアクセス制御 (RBAC) アクセス許可モデルに基づいています。 これは、Exchange で使用されるアクセス許可モデルと同じなので、Exchange に精通していなけら、セキュリティ & コンプライアンス センターでアクセス許可を付与する方法はとてもよく似ています。 ただし、Exchange の役割グループとセキュリティ コンプライアンス センターの役割グループは &メンバーシップまたはアクセス許可を共有しない点に注意してください。 どちらにも [組織の管理] 役割グループがありますが、同じではありません。 この役割グループが付与するアクセス許可およびメンバーシップは異なります。 下部に、セキュリティ センターとコンプライアンス センター&一覧があります。

## <a name="relationship-of-members-roles-and-role-groups"></a>メンバー、役割、役割グループの関係

**役割**は、一連のタスクを実行するアクセス許可を付与します。たとえば、[ケース管理] 役割は電子情報開示ケースをユーザーが扱えるようにします。

役割 **グループは** 、ユーザーがコンプライアンス センターを使用して職を実行できる一連の&です。たとえば、コンプライアンス管理者役割グループには、[ケース管理]、[コンテンツ検索]、および [組織の構成] (その他) の役割が含まれています。コンプライアンス管理者である他のユーザーが、それらのタスクを行うためのアクセス許可を必要とするからです。

コンプライアンス センターには&、ユーザーに割り当てる必要がある最も一般的なタスクと機能の既定の役割グループが含まれています。 単に個々のユーザーをメンバーとして既定 **の役割グループに** 追加する方法をお勧めします。
