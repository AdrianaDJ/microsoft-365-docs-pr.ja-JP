# <a name="create-safe-sender-lists-in-eop"></a>EOP で差出人セーフ リストを作成する

Exchange Online のメールボックスを使用している Microsoft 365 のお客様、または Exchange Online のメールボックスを使用しないもののスタンドアロンの Exchange Online Protection (EOP) をお使いのお客様の場合、EOP は、ユーザーが信頼できる送信者からメールを受信するための複数の方法を提供します。 これらのオプションには、Exchange メール フロー ルール (トランスポート ルール)、Outlook の差出人セーフ リスト、IP 許可一覧 (接続フィルター)、スパム対策ポリシーの許可されている送信者リストまたは許可されたドメインリストが含まれます。 これらのオプションは、差出人セーフ リスト _と考えて機能します_。

次のリストで、使用できる信頼できる差出人セーフ リストについて、推奨されるものを最も推奨するものから順に説明します。

1. メール フロー ルール
2. Outlook の信頼できる差出人
3. IP 許可一覧 (接続フィルター)
4. 許可された送信者の一覧または許可されたドメインリスト (スパム対策ポリシー)

メール フロー ルールによって、適切なメッセージのみを許可する柔軟性が高い。 スパム対策ポリシーの許可された送信者および許可されるドメインのリストは、送信者の電子メール ドメインが簡単にスプーフィングされているため、IP 許可一覧と同様に安全になりません。 ただし、IP 許可一覧も危険になってしかくなります_any_。その IP アドレスから送信されるドメインからの電子メールはスパム フィルター処理をバイパスするからです。

>
> - 信頼できる差出人のリストを使用 *してスパム* フィルター処理に使用した例外を、よく監視するようにします。
>
> - 信頼できる差出人リストを使用して、誤検知 (スパムとしてマークされたメール) をサポートすることもできますが、可能な場合には回避する必要がある一時的なソリューションとして、信頼できる差出人のリストの使用を検討する必要があります。 信頼できる差出人のリストを使用して誤検知を管理しないことをお勧めします。スパム フィルター処理の例外によって、組織がスプーフィングやその他の攻撃をスプーフィングするために開く可能性があるためです。 信頼できる差出人のリストを使用して誤検知を管理する場合は、いつでも大きくして、トピックの報告とファイルを [準備した状態に保管](report-junk-email-messages-to-microsoft.md) する必要があります。
>

>
> - EOP と Outlook は、メッセージの送信者を特定するために、さまざまなメッセージ プロパティを検検します。 詳細については、このトピックで後述 [するバルク メールの](#considerations-for-bulk-email) 考慮事項を参照してください。

これに対し、ブロックする差出人リストを使用して特定のソースからのメールを _ブロックするオプションもいくつか用意されています_。 詳細については、「[EOP での受信拒否リストの作成](create-block-sender-lists-in-office-365.md)」を参照してください。

## <a name="recommended-use-mail-flow-rules"></a>(推奨)メール フロー ルールを使用する

Exchange Online およびスタンドアロン EOP のメール フロー ルールでは、条件と例外を使用してメッセージを識別し、それらのメッセージに対して行う処理を指定します。 詳細については、「[Mail flow rules (transport rules) in Exchange Online](https://docs.microsoft.com/Exchange/security-and-compliance/mail-flow-rules/mail-flow-rules)」を参照してください。

次の例では、スパム フィルター処理をスキップするために、contoso.com電子メールが必要と想定しています。 これを行うには、次の設定を構成します。

## <a name="use-the-ip-allow-list"></a>Use the IP Allow List

前述のようにメール フロー ルールを使用しない場合は、次に、送信元のメール サーバー (複数の場合も) を接続フィルター ポリシーの IP 許可一覧に追加します。 詳細については [、「EOP で接続フィルターを構成する」を参照してください](configure-the-connection-filter-policy.md)。

**注**:

- 許可されている IP アドレスの数を最小限にしなければならないため、可能な場合には IP アドレスの範囲全体を使用しないようにしてください。

- コンシューマー サービス (たとえば、クライアント) または共有インフラストラクチャoutlook.comに属する IP アドレスの範囲は使用できません。

- IP 許可一覧のエントリを定期的に確認し、不要なエントリを削除します。

> [!CAUTION]
> メール フロー ルールなど、他の検証を行わない場合、IP 許可一覧のソースからの電子メールは、スパム フィルタリングおよび送信者の認証 (SPF、DKIM、DMARC) のチェックをスキップします。 これにより、フィルターが適用されない状態で受信トレイに電子メールが正常に配信される攻撃者のリスクが高くなります。

## <a name="use-allowed-sender-lists-or-allowed-domain-lists"></a>許可された送信者の一覧または許可されたドメイン一覧を使用する

望みが最も望みが少ないオプションは、スパム対策ポリシーで許可される送信者の一覧または許可されたドメイン リストを使用するオプションです。 送信者はすべてのスパム、ス *プーフィング、* フィッシング対策、および送信者の認証 (SPF、DKIM、DMARC) をバイパスしているため、可能性のある場合はこのオプションを避ける必要があります。 この方法は、一時的なテストのみに最適です。 詳細な手順については、「EOP の「 [スパム対策ポリシーを構成する」トピックを参照](configure-your-spam-filter-policies.md) してください。

このリストの最大数は約 1000 エントリです。ただし、ポータルに入力できるのは 30 のエントリのみです。 30 を超えるエントリを追加するには、PowerShell を使用する必要があります。

> [!CAUTION]
>
> - この方法では、フィルター処理されないと、受信トレイに電子メールが正常に配信された攻撃者のリスクを高くすることができます。
>
> - 許可されたドメイン リストで、自分が自分が所有しているドメイン (承認済みドメインとも呼ばれる) やよく使用されるドメイン (microsoft.com など) は使用しません。

## <a name="considerations-for-bulk-email"></a>バルク メールに関する考慮事項

標準的な SMTP 電子メール メッセージは、*メッセージ エンベロープ*とメッセージのコンテンツで構成されます。 メッセージ エンベロープには、SMTP サーバー間でのメッセージの送信と配信に必要な情報が含まれます。 メッセージのコンテンツには、総称して "*メッセージ ヘッダー*" と呼ばれるメッセージ ヘッダー フィールドと、メッセージ本文があります。 メッセージ エンベロープは RFC 5321 で説明され、メッセージ ヘッダーは RFC 5322 で表されます。 実際のメッセージ エンベロープはメッセージ送信プロセスによって生成されるもので、実際にはメッセージの一部ではないため、受信者が実際のメッセージ エンベロープを認識することはない。

- アドレス `5321.MailFrom` **(MAIL FROM** アドレス、P1 送信者、またはエンベロープの差出人とも呼ばれる) は、メッセージの SMTP 送信で使用される電子メール アドレスです。 通常、この電子メール アドレスはメッセージ ヘッダー **の Return-Path** ヘッダー フィールドに記録されます (ただし、差出人は異なる **Return-Path メール アドレスを指定** することができます)。 メッセージが配信できない場合、そのメッセージは配信不能レポート (NDR またはバウンス メッセージとも呼ばれる) の受信者です。

- From アドレスまたは P2 送信者とも呼ばれる) は From ヘッダー フィールドのメール アドレスで、メール クライアントに表示される差出人 `5322.From` のメール アドレスです。 **From** **From**

多くの場合、 `5321.MailFrom` これらの `5322.From` アドレスは同じ (個人間のコミュニケーション) とされます。 ただし、他のユーザーに代わって電子メールが送信される場合は、異なるアドレスを設定できます。 これは、ほとんどの場合、バルク メール メッセージで発生します。

たとえば、Blue Yonder Airlines がメール広告を送信するために Margie's Travel を過去にしたとします。 受信トレイで受信するメッセージには、次のプロパティがあります。

- アドレス `5321.MailFrom` は blueyonder.airlines@margiestravel.com。

- アドレス `5322.From` はblueyonder@news.blueyonderairlines.com Outlook に表示されるプロパティです。

EOP のスパム対策ポリシーで差出人セーフ リストと信頼できるドメイン リストがアドレスのみを検疫し、アドレス `5322.From` を使用している Outlook の差出人セーフ リストと同 `5322.From` 様のものです。

このメッセージがフィルター処理されるのを防ぐには、次の手順を実行できます。

- Outlook のblueyonder@news.blueyonderairlines.com送信者 `5322.From` として、ユーザーのドメイン (アドレス) を追加する。

- [メール フロー ルールを、](#recommended-use-mail-flow-rules) 電子メール アドレス、受信者 (アドレス、blueyonder@news.blueyonderairlines.com 受信者blueyonder.airlines@margiestravel.com `5322.From` 両方) からのメッセージを検索する `5321.MailFrom` 条件を指定したメール フロー ルールを使用します。

詳細については、「EOP で差出 [人セーフ リストを作成する」を参照してください](create-safe-sender-lists-in-office-365.md)。
