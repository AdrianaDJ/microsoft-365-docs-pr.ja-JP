# <a name="communication--employee-experience"></a>コミュニケーション–従業員の利便性

生産性スコアは、ユーザーが Microsoft 365 を使用する方法と、それらをサポートするテクノロジエクスペリエンスについて、組織がどのように作業を遂行するかを分析するのに役立ちます。 スコアは従業員とテクノロジの実績に対して組織のパフォーマンスを反映し、自分のスコアを自分のような組織と比較します。 コミュニケーションデータは、従業員の experience 指標の一部です。 [生産性スコアの概要](productivity-score.md)を確認します。

## <a name="why-your-orgs-communication-score-matters"></a>組織のコミュニケーションスコアが重要な理由

Microsoft は、さまざまなコミュニケーションニーズがあることを認識しています。 質問への迅速な対応を得るには、インスタントメッセージを送信することをお勧めします。 経営陣に進捗状況の更新を送信する場合は、電子メールメッセージを選択することができます。 幅広い対象ユーザーにアクセスするには、コミュニティメッセージを投稿することを選択できます。 Microsoft 365 は、すべてのユーザーのニーズに合わせて通信モードでこの柔軟性を実現します。 調査によれば、リアルタイム通信ツールを使用することで、より統一された組織を作成し、場所に関係なく従業員士気を構築しています。 [証拠を参照してください](https://go.microsoft.com/fwlink/?linkid=2127669)。

組織内のユーザーが情報を検索して共有する方法を理解するのに役立つ、電子メール、チャット、コミュニティ投稿の使用頻度を測定します。

## <a name="how-we-calculate-the-score"></a>スコアの計算方法

コミュニケーションを目的として、組織内のコミュニケーションに関連する主要な指標と、スコアを計算するためにこれらの指標を使用するスコアモデルを組み合わせた、主な洞察を提供します。

### <a name="primary-insight"></a>主要な洞察

Microsoft 365 では、複数のモードを提供することにより、すべてのユーザーのコミュニケーションスタイルを柔軟に調整することができます。 主要な洞察では、複数のモードを使用して通信する組織内のユーザーの測定を行います。

:::image type="content" source="../../media/primaryinsight.png" alt-text="コミュニケーションモードに対する主要な洞察。":::

1. **ヘッダー:** 複数の通信モードが使用されているユーザーの主要な指標である、つまり
      1. Exchange によるメールの送信
      2. Teams によるメッセージの送信
      3. Yammer でコミュニティに投稿する

これらの製品の複数が有効になっているすべてのユーザーに対する割合 (Exchange、Teams、Yammer)。

2. **本文:** 組織内のユーザーにとって、コミュニケーションのモードの柔軟性がどのように役立つかについて詳しく説明します。
3. **現在の状態の視覚化:**
1. 青い部分がヘッダーで表されるパーセンテージを表す水平バー
2. ヘッダーで表されるパーセンテージの計算に使用される (分子/分母) を強調表示します。

      - **分子**: 過去28日間に複数の通信モードを使用しているユーザーの数
      - **分母**: 過去28日以内に複数の通信製品に対して有効としてマークされたユーザーの数
6. 重要な指標の**ピアベンチマーク**値もパーセンテージとして表示されます。

- **リソースへのリンク:** ここをクリックすると、通信に関連するビデオや記事の形式ですべてのサポートコンテンツが登録されます。 これにより、コミュニケーションのためのカスタムプレイリストに移動し、その後、すべての生産性スコアカテゴリに移動することができます。
- **傾向の視覚化:** このグラフは、主な洞察力 (過去180日間に複数の通信モードを使用しているユーザーの数) の重要指標の分子の傾向を示しています。 ここで、daily の値は、過去28日間に複数の形式の通信を使用しているユーザーの数を集計したものです。

### <a name="scoring-model"></a>スコアリングモデル

スコアは、28日間のウィンドウで複数のモードを使用した通信の頻度に基づいて計算されます。

## <a name="explore-how-people-in-your-organization-communicate"></a>組織内のユーザーがどのように通信するかを調べる

このセクションでは、ページ上のすべての分析情報に関連するアクティビティのユーザーレベルの詳細を示します。 表の列は次のように記述されています。

- **Username**  は、ユーザーの電子メールアドレスです。
- **表示名**  は、ユーザーの場合は完全な名前です。
- [**最後のアクティビティの日付**ユーザーが過去28日以内の通信内のいずれかの insights に関連するアクティビティを最後に実行した日時。
- **Department:** Azure Active Directory (AAD) でユーザーに対して定義されている部署。
- **会社:** AAD でユーザーに対して定義されている会社。
- **国:** AAD でユーザーに対して定義された国
- **状態:** AAD でユーザーに対して定義されている状態。
- **市区町村:** AAD でユーザーに対して定義された市区町村。
- **Exchange:** ユーザーが過去28日以内に Exchange が有効になっている場合は True、それ以外の場合は False。
- **Teams:** ユーザーが過去28日以内に Teams に対して有効になっている場合は True、それ以外の場合は False。
- **Yammer:** 過去28日以内にユーザーが Yammer に対して有効になっている場合は True、それ以外の場合は False。
- **電子メールの送信日:** 過去28日以内にユーザーがメールを送信した日数。
- **使用日数 Teams チャット:** 過去28日間にユーザーが Microsoft Teams でチャットメッセージを送信した日数。
- **使用日数 Teams チャネル:** 過去28日間にユーザーが Microsoft Teams でチャネルメッセージを送信した日数。
- **Yammer に投稿** された日数:ユーザーが最近28日以内に Yammer に投稿した日数。
- **電子メールで @mention 使用された日数:** 過去28日以内に @mention が含まれている電子メールをユーザーが送信した日数。
