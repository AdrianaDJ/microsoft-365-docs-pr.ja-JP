# <a name="get-started-with-insider-risk-management-preview"></a>Insider リスク管理の概要 (プレビュー)

Insider リスク管理ポリシーを使用して、組織内のリスクの通知に対してアクションを実行する危険性のあるアクティビティおよび管理ツールを特定します。 組織内でのリスク管理に関して insider リスクポリシーがどのように役立つかの詳細については、「 [Microsoft 365 の「insider リスク管理](insider-risk-management.md)」を参照してください。

>[!IMPORTANT]
>Microsoft 365 insider リスク管理ソリューションは、お客様がユーザーレベルで内部ガバナンスを促進できるようにするためのテナントレベルのオプションを提供します。 テナントレベルの管理者は、組織のメンバーに対してこのソリューションへのアクセスを提供するためのアクセス許可を設定したり、Microsoft 365 コンプライアンスセンターでデータコネクタを設定して、ユーザーレベルの id をサポートするために関連データをインポートすることができます。リスクの高い活動。 お客様は、個人ユーザーに関連する洞察を認識します。雇用に関連する大きなの動作、文字、またはパフォーマンスは、管理者が計算し、組織内の他のユーザーが利用できるようにすることができます。

## <a name="before-you-begin"></a>はじめに

Insider のリスク管理を開始する前に、Microsoft 365 のサブスクリプションを確認する必要があります。 Insider リスク管理ポリシーに含まれるユーザーは、Microsoft 365 E5 コンプライアンスライセンスを所有しているか、Microsoft 365 E5 サブスクリプションに含まれている必要があります。

既存の Microsoft 365 Enterprise E5 プランを保有せずに insider リスク管理を試みる場合は、 [microsoft 365](https://docs.microsoft.com/office365/admin/try-or-buy-microsoft-365)を既存の Office 365 サブスクリプションに追加するか、Microsoft 365 Enterprise E5 の[試用版にサインアップ](https://www.microsoft.com/microsoft-365/enterprise)することができます。

Microsoft 365 組織で insider リスク管理をセットアップして使用するには、次の手順を実行します。

## <a name="step-1-required-enable-permissions-for-insider-risk-management"></a>手順 1 (必須): insider リスク管理のためのアクセス許可を有効にする

Insider リスク管理を構成および管理するには、4つのアクセス許可ロールが使用されます。 これらの構成手順を続行するには、テナント管理者が最初に**Insider リスク管理管理者**ロールを割り当てる必要があります。

| **役割** | **ロール権限** |
| ---- | ---------------- |
| **Insider リスク管理管理者** | Insider リスク管理ポリシーの作成、読み取り、更新、削除 <br> Insider リスク管理のアクセス許可と役割を作成、読み取り、更新、削除する |
| **Insider リスク管理アナリスト** | すべての insider リスク管理の通知、ケース、および通知へのアクセス |
| **Insider リスク管理の調査官** | すべてのケースについて、すべての insider リスク管理の通知、ケース、通知、およびコンテンツエクスプローラーにアクセスする |
| **Insider リスク管理ビューアー** | すべてのケースについて、すべての insider リスク管理アラート、ケース、通知、およびコンテンツエクスプローラーへの表示のみのアクセス |

コンプライアンス管理チームの構造に応じて、insider リスク管理機能を管理する役割を構成するための2つのアクセス許可オプションがあります。 Insider リスク管理を構成するときに、次のいずれかのロール管理オプションを選択します。

1. **既定の Insider リスク管理役割グループを使用**する: この役割グループを使用して、指定した管理者、アナリスト、および調査担当者のすべてのユーザーアカウントを1つのグループに追加することにより、組織の内部のリスク管理を管理します。 この役割グループには、すべての insider リスク管理の役割が含まれています。 これは、insider リスク管理をすばやく開始する最も簡単な方法であり、個別のユーザーグループに対して定義された個別のアクセス許可を必要としない組織に適しています。
2. **さまざまな管理役割に対して異なるグループを作成**します。 insider リスク管理の構成および管理のためのアクセス許可を分ける必要がある組織では、新しい役割グループを作成し、適切な役割とユーザーを新しいグループに割り当てる必要があります。 たとえば、insider リスク管理の分析と調査にさまざまなアクセス許可が必要な場合は、アナリストのグループと、調査担当者用の別のグループを作成する必要があります。 各グループについて、これらの責任に必要な役割を割り当て、グループに割り当てる必要のあるユーザーを追加します。

役割ごとに異なるグループを構成する場合は、次の表を使用して、必要な役割を各役割グループに割り当てます。

| **グループの例** | **必要な役割** |
| ---- | ---------------- |
| **Administrators** | *Insider リスク管理管理者*の役割 |
| **担当** | *Insider リスク管理アナリスト*の役割 <br> *Case Management*役割 |
| **官** | *Insider リスク管理の捜査官*の役割 <br> *Case Management*役割 |

### <a name="option-1-add-users-to-the-insider-risk-management-role-group"></a>オプション 1: ユーザーを Insider リスク管理役割グループに追加する

Insider リスク管理を構成および管理するすべてのユーザーに対して1つの役割グループを使用する場合は、次の手順を実行して、ユーザーを既定の**Insider リスク管理**役割グループに追加します。

1. Microsoft 365 [https://protection.office.com/permissions](https://protection.office.com/permissions)組織の管理者アカウントの資格情報を使用してサインインします。

2. Microsoft Office 365 セキュリティ/コンプライアンスセンターで、[**アクセス許可**] に移動します。 Office 365 で役割を表示および管理するためのリンクを選択します。

3. [ *Insider リスク管理*] 役割グループを選択して、[**役割グループの編集**] を選択します。

4. 左側のナビゲーションウィンドウで [**メンバーの選択**] を選択し、[**編集**] を選択します。

5. [**追加**] を選択し、役割グループに追加するすべてのユーザーのチェックボックスをオンにします。

6. [**追加**] を選択し、[**完了**] を選択します。

7. [**保存**] を選択して、ユーザーを役割グループに追加します。 [**閉じる**] を選択して、手順を完了します。

### <a name="option-2-create-a-new-role-group"></a>オプション 2: 新しい役割グループを作成する

さまざまな管理役割に対して別の役割グループを作成する必要がある場合は、次の手順を実行して新しいグループを作成します。

1. Microsoft 365 [https://protection.office.com/permissions](https://protection.office.com/permissions)組織の管理者アカウントの資格情報を使用してサインインします。

2. Microsoft Office 365 セキュリティ/コンプライアンスセンターで、[**アクセス許可**] に移動します。 Office 365 で役割を表示および管理するためのリンクを選択します。

3. [**作成**] を選択します。

4. [**名前**] フィールドに、新しい役割グループにフレンドリ名を付けます。 **[次へ]** を選択します。

5. [**役割の選択**] を選択し、[**追加**] を選択します。 割り当てる必要がある役割のチェックボックスをオンにします。 たとえば、このグループが insider のリスクアナリスト用である場合は、[ *Insider リスク管理アナリスト*] の役割と*ケース管理*の役割を選択し、[**追加**と**完了**] を選択します。 **[次へ]** を選択します。

6. [**メンバーの選択**] を選択し、[**追加**] を選択します。 [ポリシーの作成] と [ポリシーの一致でメッセージを管理する] を使用するすべてのユーザーおよびグループのチェックボックスをオンにし、[**追加** **] を選択します。** **[次へ]** を選択します。

7. [**役割グループの作成**] を選択します。

役割グループとアクセス許可の詳細については、「[コンプライアンスセンターのアクセス許可](../security/office-365-security/protect-against-threats.md)」を参照してください。

## <a name="step-2-optional-configure-the-microsoft-365-human-resources-data-connector"></a>手順 2 (省略可能): Microsoft 365 ヒューマンリソースデータコネクタを構成する

Insider リスク管理は、サードパーティのリスク管理および人事のプラットフォームからインポートされたユーザーデータとログデータのインポートをサポートしています。 Microsoft 365 人事 (HR) データコネクタを使用すると、ユーザーの終了と最終雇用日を含む、CSV ファイルから人事データを取得できます。 このデータは、insider リスク管理ポリシーの警告インジケーターを駆動するのに役に立ちます。また、組織内のリスク管理の範囲全体を構成するうえで重要な部分です。

組織の Microsoft 365 HR コネクタを構成する手順については、「 [hr データをインポートするためのコネクタのセットアップ](import-hr-data.md)」トピックを参照してください。 HR コネクタを構成した後、これらの構成手順に戻ります。

>[!IMPORTANT]
>*出発社員データ盗難*テンプレートを使用してポリシーを構成しようと計画している場合は、ポリシーテンプレートの完全な信号検出機能を使用するように HR コネクタを構成する必要があります。 組織に複数の HR コネクタを構成すると、insider リスク管理はすべての人事コネクタから指標を自動的に取得します。

## <a name="step-3-optional-configure-data-loss-prevention-dlp-policies"></a>手順 3 (省略可能): データ損失防止 (DLP) ポリシーを構成する

Insider リスク管理は、不要な関係者への機密情報の意図的または偶発的な公開を特定するのに役立つ DLP ポリシーの使用をサポートしています。 *データリーク*テンプレートで insider リスク管理ポリシーを構成する場合は、ポリシーに特定の DLP ポリシーを割り当てる必要があります。 このポリシーにより、機密情報のアラートインジケーターは、組織内のリスク管理の範囲全体を構成するうえで重要な要素となります。

組織の DLP ポリシーを構成する手順については、「 [dlp ポリシーの作成、テスト、およびチューニング](create-test-tune-dlp-policy.md)」を参照してください。 DLP ポリシーを構成した後、これらの構成手順に戻ります。

>[!IMPORTANT]
>*データリーク*テンプレートを使用してポリシーを構成することを計画している場合は、ポリシーテンプレートの完全な信号検出機能を使用するように、少なくとも1つの DLP ポリシーを構成する必要があります。 組織に複数の DLP ポリシーを構成する場合は、DLP ポリシーごとに内部者のリスク管理ポリシーを割り当てる必要があります。

## <a name="step-4-required-create-an-insider-risk-management-policy"></a>手順 4 (必須): insider リスク管理ポリシーを作成する

Insider リスク管理ポリシーには、割り当てられたユーザーが含まれており、通知用に構成されているリスクインジケーターの種類を定義します。 アクティビティがアラートをトリガーするには、ポリシーを構成する必要があります。

1. [Microsoft 365 コンプライアンスセンター](https://compliance.microsoft.com)で、[ **Insider リスク管理**] に移動し、[**ポリシー** ] タブを選択します。
2. [**ポリシーの作成**] を選択してポリシーウィザードを開きます。
3. [**新しい insider リスクポリシー** ] ページで、以下のフィールドに入力します。
    - [**名前 (必須)**]: ポリシーのフレンドリ名を入力します。
    - **説明 (省略可能)**: ポリシーの説明を入力します。
    - **[ポリシーテンプレートの選択 (必須)]**: ポリシーによって監視されるリスク指標の種類を定義する[ポリシー](insider-risk-management-policies.md#policy-templates)テンプレートのいずれかを選択します。

    >[!IMPORTANT]
    >*データリーク*テンプレートを選択する場合は、後でウィザードで割り当てる必要がある1つ以上の DLP ポリシーを構成する必要があります。 「*従業員データの盗難を出発*する」テンプレートを選択する場合は、ポリシーテンプレートの完全な信号検出機能を使用するように HR コネクタを構成する必要があります。

4. [**次**へ] を選択して続行します。
5. [**ユーザー** ] ページで、[**ユーザーまたはグループの追加**] を選択して、ポリシーに含めるユーザーを定義するか、[すべての**ユーザーとメールが有効なグループ**の選択] チェックボックスをオンにします。 [**次**へ] を選択して続行します。
6. [**優先度を設定するコンテンツを指定する (オプション)** ] ページで、危険なユーザーアクティビティの優先度を設定するためのソースを割り当てることができます。
    - **Sharepoint サイト**: [ **Sharepoint サイトの追加**] を選択し、優先度を設定する sharepoint 組織を選択します。 たとえば、 *"group1@contoso.sharepoint.com/sites/group1"* のようになります。
    - **機密情報の種類**: [**機密性の高い情報**の種類を追加] を選択し、優先度を設定する感度の種類を選択します。 たとえば、 *"米国の銀行口座番号"* や *"クレジットカード番号*" などです。
    - [**秘密度ラベル**]: [**感度ラベルの追加**] を選択し、優先順位を設定するラベルを選択します。 たとえば、 *"Confidential* " や *"Secret"* などです。
7. [**次**へ] を選択して続行します。
8. [**警告インジケーターの選択**] ページに、このポリシーに対して選択したテンプレートに含まれるインジケーターが表示されます。 ウィザードの最初に*データリーク*テンプレートを選択した場合は、[ **dlp ポリシー** ] ドロップダウンリストから dlp ポリシーを選択する必要があります。
9. [**監視ウィンドウの選択**] ページで、ポリシーの[監視ウィンドウの条件](insider-risk-management-policies.md#monitoring-windows)を定義します。 監視ウィンドウを必要に応じて構成します。
10. [**次**へ] を選択して続行します。
11. [**レビュー** ] ページで、ポリシーに対して選択した設定を確認します。 [**編集**] を選択してポリシー値を変更するか、[ **Submit** ] を選択してポリシーを作成してアクティブ化します。

最初の insider リスク管理ポリシーを作成するための手順を完了すると、約24時間後にアクティビティインジケーターから通知を受信することが開始されます。 このトピックの手順4のガイダンスまたは「[新しい insider リスクポリシーの作成](insider-risk-management-policies.md#create-a-new-policy)」の手順に従って、必要に応じて追加のポリシーを構成します。
