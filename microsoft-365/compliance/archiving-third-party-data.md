# <a name="archive-third-party-data"></a>サード パーティのデータをアーカイブする

Microsoft 365 では、データ コネクタを使用して、サード パーティのデータをソーシャル メディア プラットフォーム、インスタント メッセージング プラットフォーム、ドキュメントのグループ作業プラットフォームから Microsoft 365 組織のメールボックスにインポートしてアーカイブすることができます。 データ コネクタを使用してサード パーティのデータを Microsoft 365 にインポートおよびアーカイブする主な利点の 1 つは、インポート後にさまざまな Microsoft 365 コンプライアンス ソリューションを適用できるという点です。 これにより、組織の Microsoft 以外のデータが、組織に影響を与える規制と標準に確実に準拠します。

## <a name="overview-of-compliance-solutions-that-support-third-party-data"></a>サード パーティのデータをサポートするコンプライアンス ソリューションの概要

次のセクションでは、前の表に記載されているサード パーティのデータの管理に Microsoft 365 コンプライアンス ソリューションで役立ついくつかの機能について説明します。

### <a name="litigation-hold"></a>訴訟ホールド

ユーザーのメールボックス [に訴訟ホールドを](create-a-litigation-hold.md) 適用してサード パーティのデータを保持します。 ホールドを作成するときに、削除および変更されたサードパーティ データが、指定した期間保持され、メールボックスから完全に削除されるように、保持期間 (時間ベースの *保持とも呼*ばれる) を指定できます。 または、無限にコンテンツを保持することもできます (無限の *保留と呼*ばれる)、または訴訟ホールドが削除されるまで。

### <a name="ediscovery"></a>電子情報開示

Microsoft 365 の主な 3 つの電子情報開示ツールは、コンテンツ検索、コア電子情報開示、および Advanced eDiscovery です。

- **[コンテンツ検索](content-search.md)。** コンテンツ検索ツールを使用して、インポートしたサード パーティのデータをメールボックスで検索できます。 検索結果を絞り込むには、検索クエリと条件を使用して検索結果を絞り込み、検索結果をエクスポートできます。

- **[コア電子情報開示](get-started-core-ediscovery.md)(** このツールは、ケース データにアクセスできるユーザーの制御、保留の設定をユーザーのメールボックスか検索条件に一致するユーザーのメールボックスのコンテンツに配置できるケースを作成できるため、基本的な検索およびエクスポート機能に基づいています。 つまり、ユーザー メールボックスにインポートされたサード パーティ のデータを電子情報開示ホールドの状態にすることができます。

- **[Advanced eDiscovery](overview-ediscovery-20.md)。** この強力なツールでは、ケースにカストディーンを追加し、カストディーンのデータを保留にして、さらにテーマや重複検出を分析するために、カストディー語のサードパーティのデータをレビューに読み込む、コア電子情報開示のケース機能を拡張します。 レビュー セットにサード パーティのデータを読み込むと、クエリおよびフィルター処理を実行して結果セットを絞り込むことができます。

   コア電子情報開示と Advanced eDiscovery のどちらの場合も、組織の法的調査や内部調査に関連する可能性のあるサードパーティ データを管理できます。

### <a name="retention-policies"></a>アイテム保持ポリシー

保持期間の期限が [切れた後、サードパーティ](retention.md) のデータ (およびその他のメールボックス コンテンツ) を保持した後に削除するアイテム保持ポリシーをユーザーのメールボックスに適用することができます。 また、保持ポリシーを使用して、特定の期間のサードパーティのデータを削除したり、保持期間の終了時に処理レビューをトリガーしたりすることもできます。

### <a name="records-management"></a>レコード管理

Microsoft 365 [のレコード](records-management.md) 管理機能を使用して、サードパーティのデータをレコードとして宣言できます。 これは、メールボックス内のサードパーティのデータをレコードとしてマークする保持ラベルを適用するユーザーが手動で行えます。 または、サード パーティ データ内の機密情報、キーワード、コンテンツ タイプを識別して、保持ラベルを自動適用することもできます。

### <a name="communication-compliance"></a>通信コンプライアンス

通信コンプライアンス [を使用すると](communication-compliance.md) 、サードパーティのデータを調べて、組織のデータ基準に準拠しているかどうかを確認できます。 これを行うには、組織内の不適切なメッセージに対する修復アクションを検出、取得、実行します。 たとえば、不注意な言語、機密情報、および規制遵守のためにインポートするサード パーティのデータを監視できます。

### <a name="insider-risk-management"></a>インサイダー リスクの管理

選択的な人行データなどのサードパーティのデータからのシグナルは [、内部リスク](insider-risk-management.md) 管理ソリューションで使用され、組織内の危険なアクティビティを検出、調査、および処理し、その内部リスクを最小限に抑えることができます。 たとえば、人データ コネクタによってインポートされたデータは、リスク インジケーターとして使用され、従業員データのフィッテールの検出に役立ちます。

## <a name="working-with-a-microsoft-partner-to-archive-third-party-data"></a>Microsoft パートナーとの共にサード パーティのデータをアーカイブする

サードパーティのデータをインポートおよびアーカイブするもう 1 つのオプションは、組織で Microsoft パートナーと連をする方法です。 サードパーティのデータ型が、Microsoft コンプライアンス センターで利用可能なデータ コネクタでサポートされていない場合は、サードパーティのデータ ソースからアイテムを抽出するように構成するカスタム コネクタを提供できるパートナーと連シップし、サードパーティの API で Microsoft クラウドに接続してそれらのアイテムを Microsoft 365 にインポートできます。 パートナー コネクタはまた、アイテムのコンテンツをサード パーティのデータ ソースから電子メール メッセージに変換して、それを Microsoft 365 のメールボックスにインポートします。

連作業ができるパートナーの一覧およびこの方法の詳しい手順については、「 [パートナーと連の作業」を参照して、Microsoft 365 のサードパーティのデータをアーカイブしてください](work-with-partner-to-archive-third-party-data.md)。
