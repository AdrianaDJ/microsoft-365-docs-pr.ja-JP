# <a name="data-protection-impact-assessments"></a>データ保護影響評価

一般データ保護規則 (GDPR) に基づき、データ管理者は「自然人の権利や自由に高リスクを生じさせる可能性のある」処理作業に関するデータ保護影響評価 (DPIA) を準備する必要があります。Microsoft Azure または Dynamics 365 自体には、Microsoft Azure または Dynamics 365 を使用するデータ管理者が DPIA を作成する必要が生じる特性はありません。ただし DPIA が必要であるかどうかは、データ管理者が各製品を*どのように*展開、構成、使用しているかに関する状況と詳細に応じて決定します。

このドキュメントは、データ管理者に対し、DPIA が必要であるかどうかを判断し、必要な場合には DPIA に含める詳細情報を決定する上で役立つ情報を提供することを目的としています。

- [Office 365](gdpr-dpia-office365.md)
- [Azure](gdpr-dpia-azure.md)
- [Dynamics 365](gdpr-dpia-dynamics.md)
- [Microsoft サポート/プロフェッショナル サービス](gdpr-dpia-prof-services.md)

GDPR に準拠した Microsoft のデータ保護の詳細については、***Service Trust Portal*** の[コンプライアンス](https://servicetrust.microsoft.com/ComplianceManager)を参照してください。
