# <a name="insider-risk-management-policies-preview"></a>Insider リスク管理ポリシー (プレビュー)

Insider リスク管理ポリシーは、どの従業員がスコープ内にあり、どの種類のリスクインジケーターが通知用に構成されているかを決定します。 組織内のすべてのユーザーに適用されるポリシーをすばやく作成することも、ポリシーで管理する個々のユーザーまたはグループを定義することもできます。 ポリシーは、複数または特定の Microsoft Teams、SharePoint サイト、データの機密性の種類、およびデータラベルに関するポリシー条件に焦点を当てるコンテンツの優先度をサポートします。 テンプレートを使用すると、特定のリスク指標とポリシー内で割り当てられるウエイトの量が含まれ、ポリシー内の各警告トリガーの重みを効果的に決定します。 ポリシー windows では、ポリシーを適用してアクティビティに通知するための時間枠を定義し、ポリシーの有効期間を決定するために使用される期間を定義することができます。 最大ポリシー制限は、同時に5つのアクティブなポリシーになります。 ただし、必要に応じて、追加のポリシーを構成して、ポリシーをアクティブ化および非アクティブ化することができます。

## <a name="policy-dashboard"></a>ポリシーダッシュボード

**ポリシーダッシュボード**を使用すると、組織内のポリシーと、各ポリシーに関連付けられた通知の現在の状態をすばやく確認できます。

- **ポリシー名**: ポリシーウィザードでポリシーに割り当てられた名前。
- **Active alerts**: 各ポリシーのアクティブなアラートの数。
- **確認**された通知: 過去365日以内にポリシーによって発生した警告の合計数。
- **通知に対して実行**されるアクション: 過去365日に対して確認または終了した通知の合計数。
- **ポリシーの有効性**: 通知に対して行われた合計操作数によって決定されたパーセンテージ。これは、過去1年間の確認または却下された通知の合計です。
- **Active**: ケースの状態 ("はい *"* または "*いいえ*")。

## <a name="policy-templates"></a>ポリシー テンプレート

Insider リスク管理テンプレートは、ポリシーによって監視されるリスク指標の種類を定義する定義済みのポリシー条件です。 ポリシーを作成する前に、ポリシーの作成ウィザードで、各ポリシーにテンプレートを割り当てる必要があります。 新しい insider リスクポリシーを作成するときは、次のいずれかのテンプレートから選択します。

### <a name="departing-employee-data-theft"></a>従業員データの盗難からの離脱

従業員が組織を離れる場合、従業員を退職することで、通常、データの盗難に関連する特定のリスク指標があります。 このポリシーテンプレートでは、これらの指標の優先順位を示し、検出と通知をこのリスク分野に焦点を当てます。 従業員を退職する際のデータ盗難には、SharePoint Online からファイルをダウンロードする、USB ドライブなどのポータブルデバイスにファイルをコピーする、ファイルを印刷する、個人クラウドメッセージングおよびストレージサービスにデータをコピーする resignation終了日。 このテンプレートでは、これらのアクティビティに関連するリスク指標と、従業員の雇用状況に関連する方法を優先します。

>[!IMPORTANT]
>このテンプレートを使用する場合は、組織内の従業員の resignation および解雇日情報を定期的にインポートするように Microsoft 365 HR コネクタを構成する必要があります。 組織のための Microsoft 365 HR コネクタを構成する手順については、「 [HR connector を使用](import-hr-data.md)してデータをインポートする」トピックを参照してください。

### <a name="data-leaks"></a>データリーク

多くの組織でデータを保護し、データリークを防止することは、特に従業員、デバイス、およびサービスによって作成された新しいデータの急激な増大において、常に課題となります。 従業員は、サービスとデバイス間で情報を作成、保存、および共有することができ、データの漏洩をより複雑で困難にしています。 データリークが発生すると、組織外の情報を偶発的に共有したり、悪意のある目的でデータが盗まれたりする可能性があります。 このテンプレートは、不審な SharePoint Online データのダウンロード、ファイルとフォルダーの共有、USB ドライブなどのポータブルデバイスへのファイルのコピー、ファイルの印刷、個人のクラウドメッセージングおよびストレージサービスへのデータのコピーに関して、リアルタイムでの検出に優先します。

>[!IMPORTANT]
>このテンプレートを使用する場合は、組織内の機密情報を定義するために少なくとも1つのデータ損失防止 (DLP) ポリシーを構成する必要があります。 組織の DLP ポリシーを構成する手順については、「 [dlp ポリシーの作成、テスト、およびチューニング](create-test-tune-dlp-policy.md)」を参照してください。

### <a name="offensive-language-in-email"></a>電子メールの不快な言葉

不快な問題や不適切な迷惑行為を防ぐためのアクションを検出して実行することは、リスクを回避するための重要なコンポーネントです。 Microsoft 365 の不快感を持つ言語分類子は、さまざまな種類のコンプライアンス問題について、送信された電子メールメッセージを組織の Exchange Online メールボックスからスキャンできます。 これらの分類子は、人工知能とキーワードの組み合わせを使用して、スパム対策ポリシーに違反する可能性がある電子メールの言語を識別します。 このテンプレートを使用すると、これらの分類子を使用して、不適切または不快な電子メールメッセージのコンテンツを自動的に検出するポリシーをすばやく作成することができます。 Insider リスク管理では、電子メールメッセージの英語の用語と心理をスキャンする分類子を使用して、不快感を与える言葉を探します。

## <a name="monitoring-windows"></a>監視ウィンドウ

ポリシー監視ウィンドウを使用すると、insider リスク管理ポリシーテンプレートのイベントとアクティビティに基づいて、ポリシーのアクティブ化をトリガーする期間を定義できます。 選択するポリシーテンプレートに応じて、次の監視ウィンドウを使用できます。

- **スコープ内の timespan**: すべてのポリシーテンプレートで使用可能*範囲内の timespan*は、トリガーされたイベントの**後**にウィンドウがアクティブになる日数です。 このウィンドウは、ポリシーに割り当てられたすべてのユーザーに対して発生したイベントが発生した後、1 ~ 30 日間アクティブになります。 たとえば、内部者のリスク管理ポリシーを構成し、*範囲内の timespan*を30日に設定したとします。 ポリシーを構成してから数か月が経過し、ポリシーに含まれるユーザーの1人に対してトリガイベントが発生します。 トリガーイベントは、*スコープ内の timespan*をアクティブにし、トリガーイベントが発生した30日間、そのユーザーに対してポリシーをアクティブにします。
- **履歴**期間: すべてのポリシーテンプレートで使用できます。*履歴*期間は、トリガーされたイベントの**前に**ウィンドウがアクティブになる日数です。 このウィンドウでは、ポリシーに割り当てられたすべてのユーザーに対してトリガーイベントが発生するまで0から180日がアクティブになります。 たとえば、insider リスク管理ポリシーを構成し、*履歴*期間を90日に設定したとします。 ポリシーを構成してから数か月が経過し、ポリシーに含まれるユーザーの1人に対してトリガイベントが発生します。 トリガーイベントによって、*履歴*期間がアクティブになり、そのユーザーの履歴アクティビティは、トリガーイベントの前に90日間、ポリシーによって収集されます。
- [**今後の終了] ウィンドウ**: この設定は、パブリックプレビューでは表示されますが、ポリシーには適用されません。また、このソリューションの運用リリースでは削除されます。
- [**過去の終了] ウィンドウ**: この設定はパブリックプレビューで表示されますが、ポリシーには適用されません。また、このソリューションの運用リリースでは削除されます。

## <a name="create-a-new-policy"></a>新しいポリシーを作成する

新しい insider リスク管理ポリシーを作成するには、Microsoft 365 コンプライアンスセンターで**insider リスク管理**ソリューションのポリシーウィザードを使用します。

新しいポリシーを作成するには、次の手順を実行します。

1. [Microsoft 365 コンプライアンスセンター](https://compliance.microsoft.com)で、[ **Insider リスク管理**] に移動し、[**ポリシー** ] タブを選択します。
2. [**ポリシーの作成**] を選択してポリシーウィザードを開きます。
3. [**新しい insider リスクポリシー** ] ページで、以下のフィールドに入力します。
    - [**名前 (必須)**]: ポリシーのフレンドリ名を入力します。
    - **説明 (省略可能)**: ポリシーの説明を入力します。
    - **[ポリシーテンプレートの選択 (必須)]**: ポリシーによって監視されるリスク指標の種類を定義する[ポリシー](insider-risk-management-policies.md#policy-templates)テンプレートのいずれかを選択します。

    >[!IMPORTANT]
    >*データリーク*テンプレートを選択する場合は、後でウィザードで割り当てる必要がある1つ以上の DLP ポリシーを構成する必要があります。 「*従業員データの盗難を出発*する」テンプレートを選択する場合は、ポリシーテンプレートの完全な信号検出機能を使用するように HR コネクタを構成する必要があります。

4. [**次**へ] を選択して続行します。
5. [**ユーザー** ] ページで、[**ユーザーまたはグループの追加**] を選択して、ポリシーに含めるユーザーを定義するか、[すべての**ユーザーとメールが有効なグループ**の選択] チェックボックスをオンにします。 [**次**へ] を選択して続行します。
6. [**優先度を設定するコンテンツを指定する (オプション)** ] ページで、危険なユーザーアクティビティの優先度を設定するためのソースを割り当てることができます。
    - SharePoint サイト: [ **sharepoint サイトの追加**] を選択し、優先度を設定する sharepoint 組織を選択します。 たとえば、 *"group1@contoso.sharepoint.com/sites/group1"* のようになります。
    - 機密情報の種類: [**機密性の高い情報**の種類を追加] を選択し、優先度を設定する感度の種類を選択します。 たとえば、 *"米国の銀行口座番号"* や *"クレジットカード番号*" などです。
    - [秘密度ラベル]: [**感度ラベルの追加**] を選択し、優先順位を設定するラベルを選択します。 たとえば、 *"Confidential* " や *"Secret"* などです。
7. [**次**へ] を選択して続行します。
8. [**警告インジケーターの選択**] ページに、このポリシーに対して選択したテンプレートに含まれるインジケーターが表示されます。 ウィザードの最初に*データリーク*テンプレートを選択した場合は、[ **dlp ポリシー** ] ドロップダウンリストから dlp ポリシーを選択する必要があります。
9. [**監視ウィンドウの選択**] ページで、ポリシーの[監視ウィンドウの条件](insider-risk-management-policies.md#monitoring-windows)を定義します。 監視ウィンドウを必要に応じて構成します。
10. [**次**へ] を選択して続行します。
11. [**レビュー** ] ページで、ポリシーに対して選択した設定を確認します。 [**編集**] を選択してポリシー値を変更するか、[ **Submit** ] を選択してポリシーを作成してアクティブ化します。

## <a name="update-a-policy"></a>ポリシーを更新する

既存の insider リスク管理ポリシーを更新するには、Microsoft 365 コンプライアンスセンターで**insider リスク管理**ソリューションのポリシーウィザードを使用します。

既存のポリシーを管理するには、次の手順を実行します。

1. [Microsoft 365 コンプライアンスセンター](https://compliance.microsoft.com)で、[ **Insider リスク管理**] に移動し、[**ポリシー** ] タブを選択します。
2. ポリシーダッシュボードで、管理するポリシーを選択します。
3. [ポリシーの詳細] ページで、[**ポリシーの編集**] を選択します。
4. ポリシーウィザードでは、次のフィールドを編集することはできません。
    - **Name**: ポリシーのフレンドリ名
    - **[プレイブック]**: ポリシーによって監視されるリスク指標の種類を定義するために使用するテンプレート。
5. [**説明**] フィールドに、ポリシーの新しい説明を入力します。 [**次**へ] を選択して続行します。
6. [**ユーザー** ] ページで、[**ユーザーまたはグループの追加**] を選択して、ポリシーに含めるユーザーを定義するか、[すべての**ユーザーとメールが有効なグループ**の選択] チェックボックスをオンにします。 [**次**へ] を選択して続行する
7. [**優先度を設定するコンテンツを指定する (オプション)** ] ページで、危険なユーザーアクティビティに優先順位を付けるためにソースを更新します。
    - SharePoint サイト: [ **sharepoint サイトの追加**] を選択し、優先度を設定する sharepoint 組織を選択します。 たとえば、 *"group1@contoso.sharepoint.com/sites/group1"* のようになります。
    - 機密情報の種類: [**機密性の高い情報**の種類を追加] を選択し、優先度を設定する感度の種類を選択します。 たとえば、 *"米国の銀行口座番号"* や *"クレジットカード番号*" などです。
    - [秘密度ラベル]: [**感度ラベルの追加**] を選択し、優先順位を設定するラベルを選択します。 たとえば、 *"Confidential* " や *"Secret"* などです。
8. [**次**へ] を選択して続行します。
9. [**警告インジケーターの選択**] ページに、このポリシーに対して選択したテンプレートに含まれるインジケーターが表示されます。 ウィザードの最初に*データリーク*テンプレートを選択した場合は、[ **dlp ポリシー** ] ドロップダウンリストから dlp ポリシーを選択する必要があります。
10. [**監視ウィンドウの選択**] ページで、ポリシーの[監視ウィンドウの条件](insider-risk-management-policies.md#monitoring-windows)を定義します。 監視ウィンドウを必要に応じて構成します。
11. [**レビュー** ] ページで、ポリシーに対して選択した設定を確認します。 ポリシーの値を変更する場合は [**編集**] を選択し、ポリシーの変更を更新してアクティブ化する場合は [**送信**] を選択します。

## <a name="delete-a-policy"></a>ポリシーを削除する

>[!NOTE]
>ポリシーを削除しても、ポリシーによって生成されたアクティブな通知やアーカイブされた通知は削除されません。

既存の insider リスク管理ポリシーを削除するには、次の手順を実行します。

1. [Microsoft 365 コンプライアンスセンター](https://compliance.microsoft.com)で、[ **Insider リスク管理**] に移動し、[**ポリシー** ] タブを選択します。
2. ポリシーダッシュボードで、管理するポリシーを選択します。
3. ダッシュボードツールバーの [**削除**] を選択します。
4. [**削除**] ダイアログで、[**はい**] を選択してポリシーを削除するか、[**キャンセル**] を選択してダイアログを閉じます。
